# 감사 로그 조회 필터 설계

## 목표

감사 로그 화면에서 워크스페이스의 채팅 기록을 조건으로 좁혀 볼 수 있게 합니다. 운영자가 질문 내용, 문서 ID, 요청 ID, 발생 시각 범위로 기록을 빠르게 찾을 수 있도록 백엔드 API와 프론트엔드 조회 조건을 함께 확장합니다.

## 포함 범위

- `GET /v1/audit-logs/chat`에 필터 쿼리 파라미터를 추가합니다.
- 인메모리 감사 로그 저장소와 PostgreSQL 감사 로그 저장소가 동일한 필터 계약을 사용합니다.
- 프론트엔드 감사 로그 화면에 검색어, 문서 ID, 요청 ID, 시작/종료 시각, 조회 개수 입력을 추가합니다.
- 조회 조건은 모두 선택값이며, API Key와 워크스페이스 격리는 기존 방식 그대로 유지합니다.

## 제외 범위

- 관리자 전체 워크스페이스 조회
- 감사 로그 CSV 내보내기
- 저장소 전체 건수와 페이지네이션
- 감사 로그 삭제/보존 정책
- 이벤트 타입별 별도 화면

## API 설계

`GET /v1/audit-logs/chat`는 다음 선택 쿼리를 받습니다.

- `query`: 질문 또는 답변 미리보기 텍스트 검색
- `document_id`: 출처 또는 검색 대상 문서 ID 검색
- `request_id`: 요청 ID 정확 검색
- `from`: 발생 시각 시작 범위
- `to`: 발생 시각 종료 범위
- `limit`: 반환 개수, 기본값 100, 최대 200

응답 구조는 기존 `ChatAuditEventListResponse`를 유지합니다. `total`은 이번 응답에 포함된 필터링 결과 개수입니다.

## 화면 구조

좌측 조회 조건 패널에 필터 입력을 배치하고, 우측 결과 패널은 기존 카드 리스트를 유지합니다. 필터는 운영 도구 성격에 맞춰 한눈에 스캔 가능한 단순 입력 중심으로 구성합니다.
