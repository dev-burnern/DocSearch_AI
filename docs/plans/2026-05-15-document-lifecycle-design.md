# 문서 삭제와 재인덱싱 설계

## 목표

업로드된 문서를 운영 화면에서 삭제하거나 다시 인덱싱할 수 있게 합니다. 사용자는 문서 목록에서 특정 문서를 선택해 원본 파일, 벡터 청크, 메타데이터를 정리하거나, 저장된 원본을 기준으로 인덱싱 작업을 다시 실행할 수 있습니다.

## 포함 범위

- `DELETE /v1/documents/{document_id}` API를 추가합니다.
- 삭제 시 워크스페이스 소유권을 확인합니다.
- 삭제 시 MinIO/S3 원본 파일, Qdrant 벡터 청크, 문서 메타데이터를 정리합니다.
- `POST /v1/documents/{document_id}/reindex` API를 추가합니다.
- 재인덱싱 시 기존 원본 파일을 다시 읽고 기존 벡터 청크를 제거한 뒤 인덱싱 작업을 다시 큐에 넣습니다.
- 재인덱싱 결과는 문서 메타데이터에 최신 작업 ID, 상태, 청크 수로 반영합니다.
- 프론트엔드 문서 목록에 `재인덱싱`, `삭제` 액션을 추가합니다.

## 제외 범위

- 삭제 복구
- 소프트 삭제 상태 관리
- 대량 삭제와 대량 재인덱싱
- 관리자 전체 워크스페이스 문서 관리
- 비동기 작업 진행률 스트리밍

## API 설계

`DELETE /v1/documents/{document_id}`는 인증된 워크스페이스의 문서만 삭제합니다. 삭제 성공 시 문서 ID, 워크스페이스 ID, 삭제 여부를 반환합니다.

`POST /v1/documents/{document_id}/reindex`는 저장된 원본 파일을 다시 파싱하고 인덱싱 큐에 작업을 전달합니다. 응답은 갱신된 `DocumentRecord`를 반환합니다.

## 데이터 정합성

삭제는 원본 파일과 벡터 청크를 먼저 정리하고, 마지막에 메타데이터를 삭제합니다. 재인덱싱은 기존 벡터 청크를 먼저 삭제한 뒤 새 작업을 넣어 중복 청크가 남지 않게 합니다.
