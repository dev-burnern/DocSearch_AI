# 감사 로그 조회 UI 설계

## 목표

채팅 답변 생성 시 저장되는 감사 로그를 프론트엔드에서 확인할 수 있게 합니다. 사용자는 API Key를 입력하고 현재 워크스페이스의 최근 채팅 감사 이벤트를 조회해 질문, 모델, 토큰, 출처, 발생 시각을 검토할 수 있습니다.

## 포함 범위

- 앱 상단에 `채팅`과 `감사 로그` 탭을 추가합니다.
- 감사 로그 화면에 API Key 입력과 조회 버튼을 제공합니다.
- `/api/v1/audit-logs/chat`을 호출하는 프론트엔드 클라이언트를 추가합니다.
- 이벤트 목록에 질문, 답변 미리보기, 모델, 토큰 수, 검색/리랭킹 개수, 출처 파일, 발생 시각을 표시합니다.
- 조회 중, 빈 목록, 오류 상태를 화면에서 구분합니다.
- Vitest와 Testing Library로 앱 탭, API 계약, 화면 렌더링을 검증합니다.

## 제외 범위

- 관리자 권한 분리
- 기간/모델/문서별 복합 필터
- 페이지네이션
- 감사 로그 삭제와 보존 기간 정책
- CSV 내보내기

## 화면 구조

상단 헤더 아래에 탭을 두고, 기본 탭은 `채팅`으로 유지합니다. `감사 로그` 탭은 좌측에 조회 조건 패널, 우측에 이벤트 목록 패널을 둡니다. 운영 화면 성격이므로 장식보다 빠르게 훑을 수 있는 정보 배치를 우선합니다.

## 데이터 흐름

1. 사용자가 `감사 로그` 탭으로 이동합니다.
2. API Key를 입력하고 조회 버튼을 누릅니다.
3. 프론트엔드가 `X-API-Key` 헤더로 `/v1/audit-logs/chat`을 호출합니다.
4. 응답의 `events`와 `total`을 화면에 표시합니다.
5. 실패하면 백엔드 오류 메시지를 그대로 보여줍니다.
