# 관리자 권한 분리 설계

## 목표

DocSearch AI의 API Key 인증에 역할 개념을 추가해 일반 사용자 기능과 관리자 기능을 분리합니다. MVP에서는 감사 로그 조회와 CSV 내보내기를 관리자 전용으로 제한하고, 문서/검색/채팅 흐름은 기존 워크스페이스 사용자 권한으로 유지합니다.

## 포함 범위

- `DOCSEARCH_API_KEYS` 항목에 선택적 역할 필드를 추가합니다.
- API Key 형식은 `api_key|workspace_id|workspace_name(|role)`을 지원합니다.
- 역할이 생략된 기존 3필드 API Key는 일반 사용자(`member`)로 해석합니다.
- `/v1/workspace` 응답에 현재 역할을 포함합니다.
- 감사 로그 조회와 CSV 내보내기 API는 `admin` 역할만 접근할 수 있습니다.
- 일반 사용자가 관리자 API를 호출하면 403 오류를 반환합니다.
- 로컬 Compose 예시에는 관리자 키와 일반 사용자 키를 함께 제공합니다.
- 운영 환경에서는 역할 필드가 붙어도 개발 기본 API Key가 남아 있으면 준비 상태를 `not_ready`로 판단합니다.

## 제외 범위

- 사용자 로그인/세션 기반 권한 관리
- 관리자 화면 전용 UI 분리
- 여러 역할과 세부 권한 매트릭스
- 워크스페이스 간 관리자 통합 조회
- API Key 발급/회전 관리 화면

## 권한 정책

- `member`: 문서 업로드, 문서 검색, 채팅 질의응답, 본인 워크스페이스 조회 가능
- `admin`: `member` 권한에 더해 감사 로그 조회와 CSV 내보내기 가능

## 오류 계약

관리자 권한이 필요한 API에 일반 사용자가 접근하면 다음 형태의 403 응답을 반환합니다.

```json
{
  "detail": {
    "code": "AUTH_FORBIDDEN_ROLE",
    "message": "Admin role is required."
  }
}
```
