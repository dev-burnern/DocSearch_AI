# 문서 작업 화면 설계

## 목표

프론트엔드에서 문서를 업로드하고, 업로드된 문서를 기준으로 검색 결과를 확인할 수 있게 합니다. 사용자는 API Key로 워크스페이스를 인증하고, 문서 업로드 후 생성된 문서 ID와 인덱싱 상태를 확인한 뒤 검색어로 관련 청크를 조회합니다.

## 포함 범위

- 앱 상단 탭에 `문서` 탭을 추가합니다.
- 문서 화면에서 API Key, 파일 선택, 업로드 버튼을 제공합니다.
- `POST /api/v1/documents` 업로드 API를 호출합니다.
- 업로드 응답의 문서 ID, 파일명, 파서, 글자 수, 청크 수, 인덱싱 상태, 미리보기를 표시합니다.
- 독립 검색용 `POST /v1/search` 백엔드 API를 추가합니다.
- 검색 화면에서 검색어, 선택 문서 ID, 검색 개수를 입력하고 검색 결과 청크를 표시합니다.
- 업로드/검색의 로딩, 빈 결과, 오류 상태를 구분합니다.

## 제외 범위

- 업로드된 문서 목록 영속 조회
- 파일 삭제
- 검색 결과 리랭킹 UI
- 검색 결과에서 바로 채팅 질문 생성
- 권한별 관리자 UI

## API 설계

`POST /v1/search`는 API Key 인증을 사용합니다. 요청은 `query`, `document_ids`, `limit`을 받고, 응답은 검색된 청크 목록과 총 개수를 반환합니다. 내부적으로 기존 `DenseRetriever`, `RetrievalFilter`, Qdrant 저장소를 재사용합니다.

## 화면 구조

문서 탭은 좌측 업로드 패널과 우측 검색 패널로 구성합니다. 업로드 패널은 파일을 등록한 뒤 결과 메타데이터를 보여주고, 검색 패널은 검색어와 문서 ID 필터를 입력해 관련 청크를 확인합니다. 운영 도구 성격에 맞춰 카드 중첩 없이 두 개의 독립 패널로 구성합니다.
