# 업로드 문서 목록 조회 설계

## 목표

문서 업로드 이후 생성된 문서 메타데이터를 워크스페이스 단위로 조회할 수 있게 합니다. 사용자는 `문서` 탭에서 현재 워크스페이스에 업로드된 파일명, 문서 ID, 파서, 청크 수, 인덱싱 상태, 미리보기를 확인하고 검색 대상 문서를 고를 수 있습니다.

## 포함 범위

- `DocumentRecord` 메타데이터 모델을 추가합니다.
- 문서 업로드 성공 시 메타데이터 저장소에 업로드 결과를 기록합니다.
- `GET /v1/documents` 목록 API를 추가합니다.
- 목록 조회는 API Key 인증으로 확인된 워크스페이스 문서만 반환합니다.
- 로컬 기본 실행에서는 PostgreSQL 메타데이터 저장소를 사용하고, 테스트와 단독 실행을 위해 인메모리 저장소를 유지합니다.
- 프론트엔드 문서 화면에 목록 조회 버튼과 업로드 문서 리스트를 추가합니다.

## 제외 범위

- 문서 삭제와 재인덱싱
- 파일 상세 다운로드
- 목록 검색, 정렬, 페이지네이션 UI
- 관리자 전체 워크스페이스 조회
- Alembic 기반 마이그레이션 체계

## 저장 구조

문서 레지스트리는 업로드 결과에서 필요한 필드를 그대로 저장합니다.

- `document_id`: 문서 고유 ID
- `workspace_id`: 워크스페이스 필터 키
- `workspace_name`: 표시 이름
- `filename`: 원본 파일명
- `parser`: 사용된 파서
- `character_count`: 추출 텍스트 길이
- `text_preview`: 추출 텍스트 미리보기
- `storage_key`: MinIO/S3 원본 저장 키
- `indexing_job_id`: 인덱싱 작업 ID
- `indexing_status`: 인덱싱 상태
- `chunk_count`: 생성된 청크 수
- `uploaded_at`: 업로드 시각

## API 설계

`GET /v1/documents`는 `X-API-Key`로 인증된 워크스페이스의 문서 목록을 최신 업로드 순으로 반환합니다. 응답은 `documents`와 `total`을 포함합니다.

## 화면 구조

기존 문서 탭의 좌측 업로드 패널은 유지하고, 우측 검색 패널 상단에 문서 목록 영역을 추가합니다. 목록에서 문서 ID를 바로 복사할 수 있게 하여 검색 문서 ID 입력으로 이어지는 흐름을 단순화합니다.
