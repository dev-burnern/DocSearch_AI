# 관리자 화면 표시 조건 설계

## 목표

프론트엔드 앱 셸에서 API Key의 워크스페이스와 역할을 확인하고, `admin` 역할일 때만 감사 로그 탭을 표시합니다. 일반 사용자는 채팅과 문서 기능만 보고, 관리자 전용 화면은 탭 목록에서 숨깁니다.

## 포함 범위

- 앱 상단에 API Key 확인 영역을 둡니다.
- `/v1/workspace`를 호출해 `workspace_id`, `workspace_name`, `role`을 가져옵니다.
- 확인된 API Key를 채팅, 문서, 감사 로그 화면에 전달합니다.
- `role=admin`일 때만 감사 로그 탭을 표시합니다.
- `member` 역할로 확인되면 감사 로그 탭은 숨기고, 기존 선택 탭이 감사 로그라면 채팅 탭으로 전환합니다.
- API Key 확인 실패 시 역할 상태와 탭 표시를 초기화합니다.

## 제외 범위

- 로그인/세션 저장
- API Key 발급/회전 UI
- 관리자 전용 운영 상태 대시보드
- 상세 권한 매트릭스

## 화면 흐름

1. 사용자가 앱 상단에 API Key를 입력합니다.
2. `확인` 버튼을 누르면 `/v1/workspace`를 호출합니다.
3. 성공 시 워크스페이스 이름과 역할 태그를 보여줍니다.
4. `admin`이면 감사 로그 탭을 표시합니다.
5. `member`이면 감사 로그 탭을 숨깁니다.
6. 실패 시 오류 메시지를 표시하고 확인된 API Key를 제거합니다.

## 테스트 기준

- 기본 화면에서는 감사 로그 탭이 보이지 않습니다.
- admin API Key 확인 후 감사 로그 탭이 보입니다.
- member API Key 확인 후 감사 로그 탭은 계속 보이지 않습니다.
- 감사 로그 탭에 있는 상태에서 member API Key로 다시 확인하면 채팅 탭으로 이동합니다.
- API Key 확인 실패 시 오류 메시지를 표시하고 관리자 탭을 숨깁니다.
