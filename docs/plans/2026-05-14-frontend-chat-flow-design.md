# 프론트엔드 채팅 플로우 설계

## 목표

백엔드에 준비된 `/v1/chat` API를 실제 사용자가 호출할 수 있는 첫 프론트엔드 화면으로 연결합니다. 이번 단계의 핵심은 문서 기반 질문, API Key 인증, 답변 표시, 출처 확인이 한 화면에서 자연스럽게 이어지는 것입니다.

## 포함 범위

- 기존 스캐폴드 카드 화면을 채팅 작업 화면으로 교체합니다.
- API Key, 선택 문서 ID, 검색 개수, 질문 입력을 제공합니다.
- `/api/v1/chat` 경로를 통해 FastAPI 채팅 API를 호출합니다.
- 답변, 모델명, 검색 청크 수, 토큰 수, 출처 목록을 표시합니다.
- API 오류 메시지를 화면에 노출합니다.
- Vitest와 Testing Library 기반 프론트엔드 테스트를 추가합니다.

## 제외 범위

- 로그인 화면
- 문서 업로드 UI
- 감사 로그 조회 UI
- 스트리밍 답변
- 채팅 히스토리 영속화

## 화면 구조

화면은 좌측 설정 패널과 우측 답변 패널로 나눕니다. 좌측에는 API Key, 문서 ID, 검색 개수를 배치하고, 우측에는 질문 입력, 전송 버튼, 답변 결과, 출처 목록을 배치합니다. 운영 도구 성격에 맞춰 장식보다 정보 밀도와 읽기 쉬운 간격을 우선합니다.

## 데이터 흐름

1. 사용자가 API Key와 질문을 입력합니다.
2. 프론트엔드가 문서 ID 문자열을 배열로 정리합니다.
3. `ChatClient`가 `X-API-Key` 헤더와 함께 `/v1/chat` 요청 형식으로 변환합니다.
4. 응답의 답변과 출처를 화면에 표시합니다.
5. 실패 시 백엔드 `detail.message`를 우선 사용해 오류를 보여줍니다.
