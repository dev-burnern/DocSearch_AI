# 채팅 API 구현 계획

## 1단계: 테스트 기준선

- 채팅 서비스가 워크스페이스 필터로 검색을 호출하는지 검증합니다.
- 검색된 청크가 LLM 요청의 문서 컨텍스트로 들어가는지 검증합니다.
- 답변 응답에 출처, 모델명, 토큰 사용량, 검색 청크 수가 포함되는지 검증합니다.
- 검색 결과가 없을 때 컨텍스트 오류가 발생하는지 검증합니다.
- `/v1/chat`이 API Key 인증 후 응답을 반환하는지 검증합니다.

## 2단계: 서비스 경계 추가

- `ChatRequest`, `ChatResponse`, `ChatCitation`, `ChatUsage` 모델을 정의합니다.
- `ChatService`에서 검색, 컨텍스트 구성, LLM 호출, 출처 생성을 하나의 흐름으로 묶습니다.
- 컨텍스트가 없을 때 사용할 `ChatContextNotFoundError`를 정의합니다.

## 3단계: 라우터 연결

- `POST /v1/chat` 라우터를 추가합니다.
- 기존 API Key 인증 의존성을 재사용합니다.
- 기존 Dense retrieval, vLLM 클라이언트 의존성을 연결합니다.
- 컨텍스트 없음과 LLM 공급자 오류를 API 오류 응답으로 변환합니다.

## 4단계: 문서와 설정 갱신

- `.env.example`에 `CHAT_RETRIEVAL_LIMIT`를 추가합니다.
- README 현재 범위에 출처 포함 채팅 API를 반영합니다.
- 설계 문서에 요청/응답 형식과 오류 정책을 남깁니다.
