# 감사 로그 설계

## 목표

채팅 답변 생성 시점의 요청, 검색/리랭킹 결과, 모델, 토큰 사용량, 출처 정보를 추적 가능한 이벤트로 남깁니다. MVP에서는 영속화 저장소를 붙이기 전, 감사 로그 인터페이스와 인메모리 구현으로 API 계약을 먼저 고정합니다.

## 범위

- 채팅 감사 이벤트 모델 정의
- 인메모리 감사 로그 저장소 추가
- 채팅 성공 응답 이후 감사 이벤트 기록
- 워크스페이스 인증 기반 `GET /v1/audit-logs/chat` 조회 API 추가
- 답변 전문 대신 preview와 문자 수 기록

## 제외 범위

- PostgreSQL 영속화
- 감사 로그 보존 기간 정책
- 관리자 전용 권한 분리
- 실패 이벤트 기록
- 프론트엔드 감사 로그 화면

## 기록 필드

- 요청 ID, 워크스페이스 ID, 워크스페이스 이름
- 질문, 선택 문서 ID
- 검색 limit, 리랭킹 top-k, 최종 컨텍스트 청크 수
- 모델명, 토큰 사용량
- 답변 preview, 답변 문자 수
- 출처 문서 ID, 파일명, 청크 인덱스, 검색 점수, 리랭킹 점수

## 후속 작업

다음 단계에서는 감사 로그 저장소를 PostgreSQL 테이블로 옮기고, 관리자 권한과 조회 필터를 분리합니다.
