# LLM 게이트웨이 설계

## 목표

DocSearch AI의 답변 생성 단계가 특정 추론 서버 구현에 직접 묶이지 않도록, 백엔드 내부에 LLM 호출 경계를 둡니다. MVP 기준 추론 엔진은 vLLM이며, OpenAI 호환 `chat/completions` API를 사용합니다.

## 범위

- vLLM/OpenAI 호환 채팅 완료 요청 계약 정의
- 모델 프로필과 환경 변수 기반 설정 정의
- HTTP 오류, 타임아웃, 비정상 응답을 명확한 예외로 변환
- 응답 본문, 모델명, 종료 사유, 토큰 사용량을 표준 응답 객체로 반환

## 제외 범위

- 실제 채팅 API 엔드포인트
- RAG 컨텍스트 빌더와 출처 생성
- 모델 자동 선택 UI
- 스트리밍 응답

## 모델 프로필

기본 모델은 `google/gemma-4-E4B-it`로 둡니다. 사용자는 환경 변수로 vLLM 엔드포인트, 모델명, API Key, 타임아웃, 최대 토큰 수, temperature를 바꿀 수 있습니다.

## 호출 흐름

1. 서비스 계층이 `LLMRequest`를 생성합니다.
2. `VLLMClient`가 모델 프로필을 기준으로 `/chat/completions`에 요청합니다.
3. vLLM 응답을 `LLMResponse`로 검증하고 변환합니다.
4. 오류 응답이나 잘못된 응답은 `LLMProviderError`로 상위 계층에 전달합니다.
