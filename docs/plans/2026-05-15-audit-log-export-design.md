# 감사 로그 내보내기 설계

## 목표

감사 로그 화면에서 현재 조회 조건에 맞는 채팅 감사 로그를 CSV 파일로 내려받을 수 있게 합니다. 운영자는 워크스페이스별 질문, 답변 미리보기, 출처, 토큰 사용량을 외부 보고서나 점검 자료로 옮길 수 있습니다.

## 포함 범위

- `GET /v1/audit-logs/chat/export` API를 추가합니다.
- 기존 감사 로그 조회 필터를 내보내기에도 동일하게 적용합니다.
- 내보내기는 인증된 워크스페이스의 감사 로그만 포함합니다.
- CSV에는 이벤트 ID, 발생 시각, 워크스페이스, 요청 ID, 질문, 답변 미리보기, 문서 ID, 출처 파일, 모델, 토큰 수를 포함합니다.
- 프론트엔드 감사 로그 화면에 `CSV 내보내기` 버튼을 추가합니다.
- 내보내기 실패 시 기존 오류 영역에 메시지를 표시합니다.

## 제외 범위

- Excel 전용 서식 파일 생성
- 대용량 비동기 내보내기 작업
- 관리자 전체 워크스페이스 통합 내보내기
- 감사 로그 삭제 또는 보존 정책 관리

## API 설계

`GET /v1/audit-logs/chat/export`는 `GET /v1/audit-logs/chat`과 동일한 쿼리 파라미터를 받습니다. 응답은 `text/csv; charset=utf-8`이며, `Content-Disposition` 헤더로 파일명을 전달합니다.

## CSV 형식

한 이벤트는 한 행으로 내보냅니다. 여러 문서 ID나 출처 파일은 세미콜론으로 묶어 하나의 셀에 기록합니다. 질문이나 답변에 쉼표, 줄바꿈, 따옴표가 있어도 표준 CSV 인코딩으로 안전하게 보존합니다.

## 후속 단계

대량 감사 로그가 쌓이면 비동기 내보내기 작업과 관리자 권한 분리를 별도 PR에서 진행합니다.
